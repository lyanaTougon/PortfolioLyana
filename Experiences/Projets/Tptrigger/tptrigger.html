TP PostgreSQL - Triggers

1. À quoi servent les triggers ?

Les triggers permettent d'assurer une parfaite intégrité des données. Le mot "trigger" peut être traduit par "déclencheur" en français. Un trigger est un script qui se déclenche sur certains évènements de la base de données. Les triggers se définissent sur des objets de type TABLE et concernent les évènements suivants : INSERT, UPDATE, DELETE. Ils peuvent être déclenchés avant (BEFORE) ou après (AFTER) un évènement.

2. Comment ça marche ?

2.1 Préalable

Avant toute déclaration ou manipulation de trigger, il est nécessaire de déclarer au moteur PostgreSQL que l'on utilise le langage PL/pgSQL :

CREATE LANGUAGE PLPGSQL;

2.2 Création d'un trigger

La syntaxe d'un trigger est la suivante :

CREATE TRIGGER nom_trigger
{BEFORE | AFTER} {INSERT | UPDATE | DELETE}
ON nom_table
EXECUTE PROCEDURE nom_fonction();
>

2.3 Mon premier trigger

2.3.1 Création de la table

CREATE TABLE Employee(
    nom_employe TEXT,
    salaire INTEGER,
    utilisateur_dermodif TEXT
);

2.4 Définition des fonctions

CREATE FUNCTION fct() RETURNS trigger AS $fct$
BEGIN
    IF NEW.nom_employe IS NULL THEN
        RAISE EXCEPTION 'nom_employe ne peut pas être NULL';
    END IF;
    RETURN NEW;
END;
$fct$ LANGUAGE plpgsql;

2.5 Création des triggers

CREATE TRIGGER trig
BEFORE INSERT OR UPDATE
ON Employee
FOR EACH ROW
EXECUTE PROCEDURE fct();

2.6 Insertion des données

Insertion valide :

INSERT INTO Employee VALUES ('Toto', 154, 'Tata');

Insertion invalide :

INSERT INTO Employee VALUES (NULL, 154, 'Titi');

3. Conclusion

Les triggers permettent :

  • de contrôler automatiquement les données
  • d'empêcher les valeurs invalides
  • de sécuriser la base de données
